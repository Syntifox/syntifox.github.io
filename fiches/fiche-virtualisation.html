Virtualisation — TP Proxmox / VirtualBox

Mise en place d'un environnement virtualisé avec Proxmox VE et VirtualBox : création et gestion de machines virtuelles, réseaux virtuels et snapshots.

📅 2024 🏫 TP École ⏱️ 2 semaines 👤 Individuel

🎯 Contexte et objectifs

Dans le cadre du cours d'infrastructure, j'ai mis en place un hyperviseur Proxmox VE pour héberger plusieurs machines virtuelles simulant une infrastructure d'entreprise : serveur Windows, serveur Linux, client Windows et pare-feu.

Objectif : Déployer une infrastructure virtualisée complète — serveur AD Windows, serveur web Linux, routeur pfSense — sur Proxmox VE.

🛠️ Réalisation

  • Installation de Proxmox VE sur serveur physique dédié
  • Configuration du réseau virtuel (bridges, VLAN)
  • Création de VMs : Windows Server 2022, Debian 12, pfSense
  • Configuration du stockage (LVM, images ISO)
  • Mise en place des snapshots pour restauration rapide
  • Tests de haute disponibilité et de restauration
# Créer une VM via CLI Proxmox qm create 100 --name "win-server" --memory 4096 \ --cores 2 --net0 virtio,bridge=vmbr0 \ --cdrom local:iso/windows-server-2022.iso # Prendre un snapshot qm snapshot 100 snap-avant-config --description "Avant configuration AD"

✅ Résultats

L'infrastructure virtualisée est fonctionnelle. Les 3 VMs communiquent correctement via le réseau virtuel. Les snapshots permettent une restauration rapide en cas de mauvaise manipulation. L'environnement est utilisé pour tous les autres TPs du BTS.

💡 Ce que j'ai appris

La virtualisation m'a permis de comprendre comment optimiser les ressources matérielles et d'expérimenter différentes configurations sans risque. Proxmox m'a initié à la gestion d'un hyperviseur de type 1, proche des environnements professionnels.

À compléter : Captures d'écran Proxmox, schéma de l'infrastructure virtuelle, configuration réseau, performances mesurées.
