Configuration d'un pare-feu pfSense

Installation et configuration d'un pare-feu pfSense en environnement de TP : règles de filtrage, DMZ, VPN et portail captif.

📅 2024 🏫 TP École ⏱️ 1 semaine 👤 Individuel

🎯 Contexte et objectifs

Dans le cadre d'un TP de sécurité réseau, j'ai installé et configuré un pare-feu pfSense pour protéger un réseau d'entreprise simulé. L'objectif était de mettre en place une politique de filtrage réseau, une zone DMZ et un VPN pour les accès distants.

Topologie : Internet → pfSense → LAN (192.168.1.0/24) + DMZ (192.168.2.0/24) — environnement virtualisé VirtualBox.

🛠️ Configuration réalisée

  • Installation de pfSense 2.7 sur machine virtuelle
  • Configuration des interfaces WAN, LAN et DMZ
  • Mise en place des règles de filtrage (whitelist/blacklist par port et IP)
  • Création d'une zone DMZ pour isoler un serveur web
  • Configuration du VPN OpenVPN pour accès distant sécurisé
  • Mise en place du portail captif pour le Wi-Fi invité
  • Activation et analyse des logs de connexion
# Règle de filtrage — bloquer tout sauf HTTP/HTTPS depuis LAN # Interface LAN → WAN Action : Pass Protocol : TCP Source : LAN net Destination : any Port : 80, 443 # Bloquer le reste Action : Block Protocol : any Source : LAN net Destination : any

✅ Résultats

Le pare-feu est opérationnel avec les règles de filtrage configurées. La DMZ isole correctement le serveur web du LAN interne. Le VPN OpenVPN permet un accès distant chiffré. Les logs confirment le bon fonctionnement du filtrage.

💡 Ce que j'ai appris

Ce TP m'a permis de comprendre le fonctionnement d'un pare-feu stateful et les principes de la segmentation réseau. J'ai compris l'importance d'une politique de sécurité claire avant toute configuration.

À compléter : Schéma réseau, captures d'écran de l'interface pfSense, résultats des tests de filtrage, configuration VPN.
